# 计划：移除外发状态并添加状态筛选功能

## 需求分析

### 需求1：移除外发状态
当前状态流转：
- 1: 待审批
- 2: 已审批
- 3: 已拒绝
- 4: 已外发 ← 需要移除
- 5: 已验收
- 6: 已对账
- 7: 已财务审核

修改后状态流转：
- 1: 待审批
- 2: 已审批
- 3: 已拒绝
- 4: 已验收（原5）
- 5: 已对账（原6）
- 6: 已财务审核（原7）

### 需求2：为所有界面添加状态筛选功能
当前只有 ApplicationForm 和 ReportForm 有状态筛选，需要为以下界面添加：
- AuditForm（审批管理）
- AcceptanceForm（验收管理）
- ReconciliationForm（对账管理）
- FinanceAuditForm（财务审核）

## 具体修改计划

### 一、修改状态定义（ExternalProcessingApplication.cs）

**修改 Status 属性注释和 StatusText 属性：**
```csharp
// 原状态
public int Status { get; set; }  // 1:待审批, 2:已审批, 3:已拒绝, 4:已外发, 5:已验收, 6:已对账, 7:已财务审核

public string StatusText => Status switch
{
    1 => "待审批",
    2 => "已审批",
    3 => "已拒绝",
    4 => "已外发",
    5 => "已验收",
    6 => "已对账",
    7 => "已财务审核",
    _ => "未知"
};

// 修改后状态
public int Status { get; set; }  // 1:待审批, 2:已审批, 3:已拒绝, 4:已验收, 5:已对账, 6:已财务审核

public string StatusText => Status switch
{
    1 => "待审批",
    2 => "已审批",
    3 => "已拒绝",
    4 => "已验收",
    5 => "已对账",
    6 => "已财务审核",
    _ => "未知"
};
```

### 二、修改 AcceptanceForm.cs（验收管理）

1. **移除外发按钮（BtnDispatch）**
2. **修改状态判断逻辑：**
   - 原：已审批(2) → 外发(4) → 验收(5)
   - 新：已审批(2) → 直接验收(4)
3. **修改 LoadApplications 加载的状态：**
   - 原：加载已审批(2)、已外发(4)、已验收(5)
   - 新：加载已审批(2)、已验收(4)
4. **修改验收按钮逻辑：**
   - 原：检查状态是否为已外发(4)
   - 新：检查状态是否为已审批(2)
   - 更新状态为已验收(4)
5. **修改反审按钮逻辑：**
   - 原：已验收(5)反审回到已外发(4)
   - 新：已验收(4)反审回到已审批(2)
6. **添加状态筛选功能**

### 三、修改 ReconciliationForm.cs（对账管理）

1. **修改 LoadApplications 加载的状态：**
   - 原：加载已验收(5)、已对账(6)

2. **修改对账按钮逻辑：**
   - 原：检查状态是否为已验收(5)，更新为已对账(6)
   - 新：检查状态是否为已验收(4)，更新为已对账(5)
3. **修改反审按钮逻辑：**

   - 新：已对账(5)反审回到已验收(4)

### 四、修改 FinanceAuditForm.cs（财务审核）

### 八、修改数据库中的现有数据

如果数据库中已有数据，需要执行 SQL 脚本更新状态值：
```sql
-- 将原状态 5,6,7 降级为 4,5,6
-- 原状态 4(已外发)的数据改为 2(已审批)或直接改为 4(新已验收)根据业务判断
UPDATE ExternalProcessingApplication 
SET Status = CASE 
    WHEN Status = 5 THEN 4  -- 已验收
    WHEN Status = 6 THEN 5  -- 已对账
    WHEN Status = 7 THEN 6  -- 已财务审核
    WHEN Status = 4 THEN 2  -- 原已外发改为已审批（或者根据业务改为4已验收）
    ELSE Status
END
WHERE Status IN (4, 5, 6, 7);
```
