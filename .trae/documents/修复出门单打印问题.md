# 计划：修复出门单打印问题

## 问题分析

### 问题1：数量都为零
**原因分析：**
- 申请管理界面没有数量字段
- 明细数据存储在 `ExternalProcessingApplicationDetails` 表中
- 当前申请编辑表单没有录入明细数据的功能

**解决方案：**
- 在申请编辑表单中添加数量字段
- 或者添加明细数据录入功能（DataGridView）
- 主表 `ExternalProcessingApplication` 添加 `TotalQuantity` 字段便于快速查看

### 问题2：审批日期和底部内容距离太远
**原因分析：**
- `GatePassPrintDocument.cs` 第89行：`y = pageBounds.Height - 150;`
- 底部内容从页面高度减去150的位置开始，导致与审批日期之间有大段空白

**解决方案：**
- 根据实际内容动态计算底部位置
- 在打印完审批日期后，根据当前y位置继续打印底部内容，而不是固定从底部计算

### 问题3：备注和"此联由门卫留存"重叠
**原因分析：**
- 第100-101行打印"备注："和横线
- 第104行打印"此联由门卫留存"在 `pageBounds.Height - 50`
- 如果备注位置计算不当，会重叠

**解决方案：**
- 根据实际内容动态计算页脚位置
- 确保页脚在备注下方有足够间距

### 问题4：审批人显示"当前用户"
**原因分析：**
- `AuditEditForm.cs` 第186行：`AuditorName = "当前用户"`
- 审批时硬编码了"当前用户"四个字

**解决方案：**
- 需要将当前登录用户信息传递到 AuditEditForm
- 从 MainForm -> AuditForm -> AuditEditForm 传递用户

### 二、修改打印文档布局（GatePassPrintDocument.cs）

**动态计算底部位置：**
```csharp
// 打印审批日期后，根据当前y位置继续
y += lineHeight + 10;
g.DrawString($"审批日期：{_model.AuditDate:yyyy-MM-dd}", headerFont, Brushes.Black, 50, y);

// 动态计算底部起始位置 - 在审批日期下方30像素开始
y += lineHeight + 30;

// 打印分隔线
g.DrawLine(Pens.Black, 50, y, pageBounds.Width - 50, y);
y += 15;

// 打印出门时间、签字等
g.DrawString("出门时间：_______年_____月_____日 _____时_____分", normalFont, Brushes.Black, 50, y);
y += lineHeight + 10;

g.DrawString("门卫签字：__________________", normalFont, Brushes.Black, 50, y);
g.DrawString("经办人签字：__________________", normalFont, Brushes.Black, 300, y);
y += lineHeight + 10;

// 打印备注
g.DrawString("备注：", headerFont, Brushes.Black, 50, y);
g.DrawLine(Pens.Black, 90, y + 18, pageBounds.Width - 50, y + 18);
y += lineHeight + 10;

// 打印页脚 - 确保在备注下方
var footerY = y + 10;
// 确保页脚不会超出页面
if (footerY > pageBounds.Height - 30)
{
    footerY = pageBounds.Height - 30;
}
g.DrawString("此联由门卫留存", smallFont, Brushes.Gray, 50, footerY);
g.DrawString($"打印时间：{DateTime.Now:yyyy-MM-dd HH:mm:ss}", smallFont, Brushes.Gray,
    pageBounds.Width - 200, footerY);
```
