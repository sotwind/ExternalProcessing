# 申请管理默认申请人功能计划

## 需求描述
在申请管理模块中，添加新申请时，申请人字段默认取当前登录用户的名字。

## 现状分析
1. **ApplicationEditForm.cs** - 申请编辑/新增窗体
   - 无参构造函数用于新增申请，此时 `_isEdit = false`
   - 当前新增时申请人字段 `TxtApplicantName` 为空

2. **ApplicationForm.cs** - 申请管理主窗体
   - 点击"新增"按钮时创建 `ApplicationEditForm()` 实例
   - 没有传递当前用户信息

3. **MainForm.cs** - 主窗体
   - 通过构造函数接收 `User currentUser` 对象
   - 包含当前用户的 `Username` 属性

4. **User.cs** - 用户模型
   - 包含 `UserID` 和 `Username` 属性

## 实现方案

### 方案：修改 ApplicationEditForm 支持传入当前用户

#### 步骤1：修改 ApplicationEditForm 构造函数
- 添加 `User` 类型的参数到无参构造函数（或创建新的构造函数）
- 在新增模式下，自动将当前用户的用户名填充到申请人字段

#### 步骤2：修改 ApplicationForm 传递当前用户
- 修改 ApplicationForm 接收当前用户对象
- 在创建 ApplicationEditForm 时传入当前用户

#### 步骤3：修改 MainForm 传递当前用户到 ApplicationForm
- 修改 MainForm 中 BtnApplication_Click 方法，将当前用户传递给 ApplicationForm

## 具体修改内容

### 1. ApplicationEditForm.cs
- 修改无参构造函数，添加 `User currentUser` 参数
- 在构造函数中设置 `TxtApplicantName.Text = currentUser.Username`

### 2. ApplicationForm.cs
- 添加私有字段 `private readonly User _currentUser;`
- 修改构造函数接收 `User currentUser` 参数
- 修改 BtnAdd_Click 方法，创建 ApplicationEditForm 时传入 `_currentUser`

### 3. MainForm.cs
- 修改 BtnApplication_Click 方法，创建 ApplicationForm 时传入 `_currentUser`

## 注意事项
- 编辑模式下仍需保持原有逻辑，从申请记录中加载申请人
- 确保新增时申请人字段默认填充且可编辑（用户可能需要修改）
